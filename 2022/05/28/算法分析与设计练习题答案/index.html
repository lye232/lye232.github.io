算法设计与分析复习题 (更新中)

# 一、名词解释：

  1. 算法：

    算法是规则的有限集合，是为解决特定问题而规定的一系列操作。

  2. 程序：

    程序是具体语言规则的描述。

  3. 递归：

    在定义自身的同时，出现对自身直接或间接的调用。

  4. 子问题的重叠性质：

    用递归算法计算最优值时，每次计算产生的子问题并不总是新问题，有些子问题被重复计算多次。

# 二、简答：

  1. 递归算法分析有哪些方法？

    替换法、递归树法、主方法。

  2. 试述分治法的基本思想。

    将一个大规模问题分割成一些规模较小的相同问题，以便分而治之。

  3. 一个算法通常具有哪些特性？

    有限性、确定性、可行性、输入和输出。

  4. 算法分析的目的是什么？

    分析算法效率以求改进。

  5. 算法的时间复杂性与问题的什么因素相关？

    问题的规模大小。

  6. 算法的渐进时间复杂性的含义？

    问题规模 n 趋于无穷大，时间复杂度 T (n) 的数量级称为渐进时间复杂度。

  7. 最坏情况下的时间复杂性和平均时间复杂性有什么不同？

    前者是解决同类问题的最大耗费，后者是耗费的平均值。

  8. 简述动态规划算法求解的基本要素。

    最优子结构、无后效性、子问题重叠性。

  9. 简述贪心算法与动态规划算法的异同。

    共同点：两者都具有最有子结构。

    不同点：动态规划最终求解的是整体最优解，贪心算法不一定求出整体最优解，一定求出近似最优解。

  10. 贪心算法求解的问题主要具有哪些性质？

    贪心选择性和最优子结构性质。

  11. 贪心算法的基本思想？

    从问题的某一初始解出发，向给定的目标推进，每次推进做一个当时看似最优的贪心选择，不断地把问题示例归纳为更小的子问题，并期望通过局部最优选择产生出一个全局最优解。

  12. 为什么要分析最坏情况下的算法时间复杂性？

    最坏情况下的时间复杂度决定算法的优劣，并且最坏情况下的时间复杂度较平均复杂度更有操作性。

  13. 简述渐进时间复杂性上界的定义。

    若存在一个正常数 C 和 n0，对所有的 n>n0，都有 f (n)<=C*g (n)，记作 f (n)=Ω(g (n))，即 f 的阶高于 g 的阶。

  14. 阐述归并排序的分治思路。

    将数组一分为二，分别对每个集合单独排序，然后将已排序的两个序列归并成一个含 n 个元素的分好类的序列。如果分割后子问题仍很大，则继续分治直到一个元素。
